# [LKPR-69] Agent hook auto-capture — lightweight capture of agent decisions without LLM

## Problem

Lorekeeper currently requires the agent to explicitly call `lore_remember` or `lore_insert` to save memories. This works well for intentional capture but misses:

- **Spontaneous decisions** — agent figures something out but doesn't think to save it
- **Passive users** — not every user will remember to prompt their agent to save
- **First impressions** — new users who install Lorekeeper but haven't trained their agent workflow yet get zero benefit until they explicitly use it

Competitors (agentmemory) solve this with auto-capture hooks. But their approach stores everything, creating noise. Lorekeeper's quality loop is the differentiator — we can capture _and then filter_, rather than capturing nothing.

## Solution

A lightweight hook layer that sits between the agent and Lorekeeper. It captures raw agent events (tool calls, decisions, file writes) and passes them through a fast, no-LLM classifier:

```
hook captures raw event → classifier (keep or discard?) →
if keep: lore_remember(title, content) →
feedback loop naturally decays noise over time
```

### Classification (no LLM needed)

Three tiers, starting simple:

**Tier 1 — Heuristic rules (~50 LOC)**
Pattern match for high-signal language:

- Decision markers: "we decided", "chose X over Y", "root cause is", "the architecture is"
- Discovery markers: "found that", "discovered", "turns out"
- Preference markers: "prefer", "standard is", "convention is"

**Tier 2 — Lightweight classifier (if Tier 1 insufficient)**
Small distilled BERT model trained on "keep vs discard" binary. Training set generated by:

1. Running Tier 1 on real sessions
2. Using the feedback loop to validate — memories that get `useful=false` are negative examples
3. Retraining periodically as feedback accumulates

**Tier 3 — Feedback loop-driven self-improvement**
The classifier learns from lore_update feedback. If a captured memory gets consistently low scores, the hook adjust its capture threshold for similar events.

### Integration points

- **MCP hook** — subscribe to agent tool call events, capture output + context
- **Session end hook** — batch-process session transcripts, extract key decisions
- **Manual override** — agent can still call `lore_remember` for explicit capture, bypassing the filter

## Why this plays to our strengths

Agentmemory captures _everything_. Lorekeeper captures _selectively_, then the feedback loop further refines. Ours gets better with use. Theirs gets noisier.

## Acceptance Criteria

- [ ] Hook captures agent events (tool calls, decisions) without blocking the agent
- [ ] Tier 1 heuristic classifier runs in <10ms per event (no LLM)
- [ ] Captured events pass through `lore_remember` with default score
- [ ] Feedback loop (`lore_update`) naturally decays false positives
- [ ] Agent can still call explicit `lore_remember` as override
- [ ] Feature is opt-in (disabled by default, env var to enable)
