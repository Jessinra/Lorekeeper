# Lorekeeper Manifesto

**We built the memory server we wanted to use. We're sharing it in case you want it too.**

Lorekeeper is local-first, self-improving, and free to run forever. One pip install, no cloud
dependency, no API keys, no infrastructure to maintain. Your agent's memories stay on your machine
and get better the more you use them — weak memories fade, strong ones rise.

It was built by AI agents using AI agents. The development cycle is a working demo of what it does:
agents build a feature, use Lorekeeper to capture what they learned, search those memories next
session, and build the next feature with the context already there. That feedback loop shaped every
design decision — the tool schemas, the search weights, the quality thresholds. When the agents that
build it depend on it, the incentives are right.

We're honest about trade-offs. The ~1.4GB embedding model is real. The benchmark data is pending.
The cloud offering doesn't exist yet. We'd rather you know upfront than discover it later.

Open source is how knowledge compounds. The memory tooling ecosystem is young and growing fast —
mem0, agentmemory, basic-memory, and others are each solving a different slice of a hard problem.
More great tools means more developers can rely on agents, and that benefits everyone. We're grateful
to be part of that conversation, not in spite of the competition but because of it.

**Don't burn bridges. Tech ecosystems thrive when there's a lot of contribution from open source and
adoption.**

Simple to install. Honest about trade-offs. Better with every session.
