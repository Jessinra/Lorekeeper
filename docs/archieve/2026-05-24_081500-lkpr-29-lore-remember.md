### 2. `orchestrator.py` — `remember()` method

Pseudocode:

```
def remember(self, thought: str) -> dict:
    # Auto-extract fields
    title = first ~80 chars of thought, ending at sentence boundary (.?!)
    description = first 60 chars of title (or "" if title ≤ 60 chars)
    score = 7.0

    # Call _insert_one_memory (existing dedup path)
    result = self._insert_one_memory({"title": title, "description": description, "content": thought, "score": score}, force=False)

    # Handle duplicate
    if "duplicate" in result:
        return {"id": result["duplicate"]["existing_memory"]["id"], "title": title, "linked_to": None}

    lore_id = result["inserted"]["id"]

    # Auto-link: query Chroma for nearest neighbor > 0.75
    sem_hits = self._engine.search(thought, limit=2)  # get 2 to skip self
    linked_to = None
    for hit in sem_hits:
        if hit["lore_id"] != lore_id and hit["score"] >= 0.75:
            self._store.insert_link(source_memory_id=lore_id, target_memory_id=hit["lore_id"], relation_type="related_to", reason=f"auto-linked from lore_remember: {hit['score']:.2f}")
            linked_to = {"id": hit["lore_id"], "score": round(hit["score"], 4)}
            break

    self._rebuild_kw()
    return {"id": lore_id, "title": title, "linked_to": linked_to}
```

**Title extraction function** (module-level helper):

```python
def _extract_title(thought: str) -> str:
    """First ~80 chars, ending at sentence boundary if possible."""
    max_len = 80
    trimmed = thought.strip()
    if len(trimmed) <= max_len:
        return trimmed
    # Find last sentence boundary within first max_len chars
    chunk = trimmed[:max_len]
    for end_char in ".!?":
        idx = chunk.rfind(end_char)
        if idx > max_len // 2:  # at least half the max length
            return chunk[:idx + 1].strip()
    # No good boundary — break at last word
    idx = chunk.rfind(" ")
    if idx > 0:
        return chunk[:idx].strip()
    return chunk
```

**Note:** `_rebuild_kw()` call needed because after the single-memory insert, the KW index is stale. `insert()` batch rebuilds after all inserts. For `remember()` single-shot, we rebuild after.

### 4. `server.py` — register MCP tool

```python
@mcp.tool(name="lore_remember")
async def lore_remember(thought: str) -> dict:
    """Fast one-shot memory insert. Pass a thought, get a memory with auto-title."""
    try:
        return handle_remember(get_service(), thought)
    except Exception:
        log.exception("lore_remember_failed", thought=thought[:80])
        raise
```

## Edge cases

- Empty thought string → handled by title extraction (trimmed empty → insert still works, title is "")
- Very long thought → only first 80 chars used for title, full content stored verbatim
- Self-match in auto-link → filter out by `lore_id != hit["lore_id"]`
- Duplicate title → `_insert_one_memory` already handles exact title match as definitive duplicate
- Chroma search returns 0 results → `linked_to` stays None
