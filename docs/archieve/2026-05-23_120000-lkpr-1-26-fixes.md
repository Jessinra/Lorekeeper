## LKPR-1: Prevent duplicate reflection on already-processed sessions

### Confirmed Root Cause

`submit_reflection` in `orchestrator.py` does **not** check if `session_id` already exists
before proceeding. The flow is:

1. `lore_reflect` (server.py:130) → `svc.submit_reflection(...)` (orchestrator.py:389)
2. `submit_reflection` immediately generates a fresh `reflection_id = uuid4()`
3. Calls `_store.insert_reflection(...)` → plain `INSERT INTO reflections ...` — no ON CONFLICT guard
4. Calls `_store.upsert_session(...)` → `INSERT ... ON CONFLICT(session_id) DO UPDATE` — idempotent for sessions, but overwrites `reflection_id` column with the new UUID

Result: every duplicate call creates a **new orphaned reflection row** and overwrites the
session's `reflection_id` pointer. The `lore_processed_sessions` API returns session IDs from
`SELECT session_id FROM sessions` — it correctly de-dups sessions — but the duplicate
**reflections** are never cleaned up and the original reflection pointer is lost.

`_store.get_session(session_id)` exists and can be used for the guard.

### Fix

In `orchestrator.py::submit_reflection`, check `self._store.get_session(session_id)`.
If a row exists, return an idempotent no-op response including the existing reflection data.

## LKPR-26: lore_insert returns unhelpful 'title' error

### Root Cause

`_insert_one_memory` in `orchestrator.py:133` does `title = m["title"]` unconditionally.
Missing `title` raises `KeyError: 'title'`, caught by outer `except Exception as e` in
`insert()`, stringified as `"'title'"` — unhelpful.

### Fix

1. `orchestrator.py` — validate `"title" in m` before access; raise
   `ValueError("memory dict missing required field: 'title'")` if absent
2. `server.py` — add docstring to `lore_insert` documenting required fields
