# LKPR-99 — Link Suggestion Sweep Engine

**Date:** 2026-06-20
**Ticket:** backlogs/ready/LKPR-99-link-suggestion-sweep-engine.md
**Approach:** New DB table + focused store + orchestrator sweep method + internal scheduler daemon thread

---

## Summary of Change

Add a `link_suggestions` DB table that stores candidate link pairs generated by the existing `LinkCandidateGenerator`. A sweep function iterates all active memories, runs the scorers, and records candidates — high-confidence ones get a `confidence='high'` tag.

An internal scheduler daemon thread polls every 5 minutes, checks `config_overrides["sweep_next_run_at"]`, and fires the sweep when the timer elapses. No external cron, no piggyback on request handlers.

**No real `memory_links` rows are created by the sweep.** All suggestions are purely in the suggestions table, waiting for future agent review tools (LKPR-100) or dashboard (LKPR-101). The stored scores and confidence labels accumulate as labelled data for training a future classifier to learn the right auto-accept threshold.

No changes to the existing `LinkCandidateGenerator` or its scorers — the sweep just calls `generator.generate(memory_id)` in a loop and writes results. LKPR-67 link types are already the canonical types in the codebase.

- **Add `sweep_links` method:**

  ```python
  def sweep_links(self) -> dict[str, Any]:
      """Iterate all active memories, generate candidates, tag and write suggestions.

      No real memory_links rows are created. Candidates above the high-confidence
      threshold get confidence='high'; others get confidence='standard'.
      Rejected and already-linked pairs are skipped.

      Returns stats: memories_scanned, candidates_generated,
      high_confidence, standard, skipped_rejected, skipped_linked, expired_pruned.
      """
  ```

  **Algorithm:**

  1. Load all active memories via `_all_memories(include_deleted=False)`
  2. Pre-load rejected pairs set via `self.suggestions.rejected_pairs()`
  3. Pre-load existing link pairs via `self.links.all_links()` (both directions)
  4. For each memory:
     a. `candidates = self._link_candidate_generator.generate(memory_id)`
     b. For each candidate with `weighted_score >= link_score_threshold`:
     - Skip if already linked (check existing pairs)
     - Skip if previously rejected (check rejected_pairs)
     - Set `confidence = 'high'` if `weighted_score >= suggest_high_confidence_score`
     - `upsert_suggestion(...)` with all score components + confidence tag
  5. Prune expired suggestions via `self.suggestions.prune_expired(settings.suggest_ttl_days)`
  6. Return stats dict

- **No `_accept_suggestion` helper** — no auto-accept, no type heuristics.

- ~80 lines added.

### 5. `src/lorekeeper/scheduler.py` (NEW)

Internal daemon scheduler — runs in a background thread inside the MCP server process.

```python
"""Internal sweep scheduler — daemon thread, no external cron needed.

Checks config_overrides["sweep_next_run_at"] every poll interval seconds.
If the timer has elapsed, fires sweep_links() and updates next_run_at.
Persists across server restarts via config_overrides table.
"""

import logging
import threading
from datetime import UTC, datetime, timedelta
from time import sleep
from typing import TYPE_CHECKING, Any

if TYPE_CHECKING:
    from lorekeeper.services.orchestrator import MemoryService

log = logging.getLogger(__name__)

class SweepScheduler:
    """Daemon thread that checks sweep timer and fires on schedule."""

    def __init__(
        self,
        svc: "MemoryService",
        interval_hours: int = 12,
        poll_seconds: int = 300,
    ) -> None:
        self._svc = svc
        self._interval = timedelta(hours=interval_hours)
        self._poll = poll_seconds
        self._stop = threading.Event()
        self._thread = threading.Thread(target=self._loop, daemon=True)

    def start(self) -> None:
        self._thread.start()

    def stop(self) -> None:
        self._stop.set()

    def _loop(self) -> None:
        while not self._stop.wait(self._poll):
            try:
                raw = self._svc.config.get("sweep_next_run_at")
                if raw is not None:
                    next_run = datetime.fromisoformat(raw)
                    if datetime.now(UTC) < next_run:
                        continue  # Not time yet
                # First run (no key) or timer elapsed
                stats = self._svc.sweep_links()
                next_time = datetime.now(UTC) + self._interval
                self._svc.config.set("sweep_next_run_at", next_time.isoformat())
                log.info("sweep_completed", stats=stats)
            except Exception:
                log.exception("sweep_failed")
```

- `ConfigStore.get(key)` and `ConfigStore.set(key, value)` already exist from dashboard config patterns — no new config store methods needed.
- Fresh DB (no `sweep_next_run_at` key) → fires the first sweep on the first poll cycle.
- On failure: logs exception, continues loop (doesn't crash the server).
- ~55 lines.

## Edge Cases & Design Decisions

| Concern                     | Decision                                                                                                            |
| --------------------------- | ------------------------------------------------------------------------------------------------------------------- |
| Self-link prevention        | `generate()` already filters `self` (skips `lore_id == source_lore_id`)                                             |
| FK cascade on memory delete | `ON DELETE CASCADE` on both FKs — orphaned suggestions auto-clean                                                   |
| Concurrent sweep + insert   | New memories added during sweep won't be evaluated until next sweep — acceptable                                    |
| Large DB (100k memories)    | Sweep is O(n) over active memories. Each memory calls `generate()` which is O(k) (cosine pre-filter). No LLM calls. |
| Rejected pair persistence   | Rejected stays in DB with `status='rejected'` — future sweeps skip via `rejected_pairs()` query                     |
| Canonical pair ordering     | `min(id1, id2)` as source prevents both-direction duplicates                                                        |
| No real links from sweep    | Suggestions table only — keeps pipeline stateless and reversible                                                    |
| Training data accumulation  | Stored scores + confidence labels form a labelled dataset for future classifier training                            |
| Scheduler crash             | Exception caught, logged, loop continues — doesn't crash the server                                                 |
| Fresh DB (no next_run_at)   | First sweep fires immediately on first poll cycle                                                                   |
| Server restart              | `next_run_at` persists in `config_overrides` — scheduler picks up where it left off                                 |
| Dry-run mode                | Available via `--dry-run` flag on `sweep-links.py` — runs logic, skips all writes                                   |
