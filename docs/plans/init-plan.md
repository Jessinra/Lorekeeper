## Architecture Decision

Two stores working together:

```
Mem0 + Chroma (vector)             SQLite sidecar (metadata + graph)
──────────────────────             ─────────────────────────────────
• Embeddings (384-dim)             • Memory metadata (score, confidence,
• Semantic ANN search                soft_deleted, usage_count, timestamps)
• Source of truth for vectors      • All MemoryLink rows (knowledge graph)
                                   • BM25 index rebuild source
```

Mem0 handles vector storage + retrieval. SQLite handles everything Mem0 doesn't (metadata, links, quality signals). The canonical `lore_id` UUID lives in Mem0's metadata field so it stays stable across re-indexing.

The MCP API surface is **identical** to the old server — same three tools, same schemas, same skill prompts work unchanged.

```python

LORE_USER_ID = "lorekeeper"  # single-tenant; Mem0 requires a user_id namespace

def build_mem0(chroma_path: Path) -> Memory:
    config = {
        "vector_store": {
            "provider": "chroma",
            "config": {
                "collection_name": "lorekeeper",
                "path": str(chroma_path),
            },
        },
        "embedder": {
            "provider": "huggingface",
            "config": {
                "model": "sentence-transformers/all-MiniLM-L6-v2",
            },
        },
        # LLM layer disabled: lore_insert stores text verbatim (infer=False on every add())
        # No API key required.
    }
    return Memory.from_config(config)
```

**Key Mem0 calls:**

| Method | Use |
|--------|-----|
| `m.add(text, user_id=LORE_USER_ID, metadata={"lore_id": id, ...}, infer=False)` | Insert verbatim (no LLM extraction) |
| `m.search(query, user_id=LORE_USER_ID, limit=200)` | Semantic candidates for hybrid re-rank |
| `m.get_all(user_id=LORE_USER_ID)` | BM25 index rebuild fallback |

> **CRITICAL**: `lore_id` (our UUID) is stored in Mem0's metadata. Mem0 assigns its own internal id. All app logic uses `lore_id` as the canonical identifier.

## Service Implementations

### Confidence EMA (`services/feedback.py`)

Exact port of the TypeScript algorithm:

```python
def compute_running_confidence(
    current: float | None,
    count: int,
    new_value: int,
    window: int,
) -> float:
    prev = current if current is not None else float(new_value)
    effective_count = min(count + 1, window)
    return prev + (new_value - prev) / effective_count

def apply_score_delta(
    current: float,
    useful: bool,
    confidence: int | None,
    settings: Settings,
) -> float:
    if useful:
        mult = (confidence / 10.0) if confidence is not None else 1.0
        return min(settings.score_max, current + settings.score_bump_up * mult)
    mult = ((11 - confidence) / 10.0) if confidence is not None else 1.0
    return max(settings.score_min, current - settings.score_bump_down * mult)

def should_soft_delete(useful: bool, confidence: int | None, threshold: int) -> bool:
    return (not useful) and (confidence is not None) and (confidence <= threshold)
    # Note: once soft_deleted=True it NEVER reverts, even on future useful=True feedback
```

```python
combined_score = (
    settings.w_semantic * semantic_score +           # 0.45
    settings.w_keyword  * keyword_score +            # 0.30
```

> **Semantic score probe**: Chroma returns either similarity (higher=better) or distance (lower=better) depending on version. At startup, add a known query and check if score increases or decreases with relevance. Build `_normalize_semantic()` accordingly. This is the #1 risk for the rebuild.

## Risks & Mitigations

| Risk | Mitigation |
|------|-----------|
| **Chroma returns distance vs similarity ambiguity** | Probe at startup with known query; log the scale; unit-test `_normalize_semantic()` |
| **BM25 (rank-bm25) ≠ Lunr numerically** | Top-1 normalization absorbs absolute-scale differences; parity test verifies ranking correlation ≥ 0.85 Spearman |
| **Mem0 `infer=False` not supported in older version** | Pin `mem0ai>=0.1.100`; fallback: stub LLM that no-ops |
| **Mem0 internal dedup silently drops inserts** | Verify with post-insert count check; `infer=False` should bypass |
| **First-run model download (~90 MB)** | Document in README; pre-download in setup script |
| **Migration not transactional across Mem0 + SQLite** | Run twice with idempotency guard; verify counts before flipping settings.json |
| **stdout pollution from Mem0/Chroma logs** | Set `MEM0_LOG_LEVEL=ERROR`; redirect all internal loggers to stderr at startup |
| **Two servers using same LORE_DATA_DIR simultaneously** | Document single-instance requirement; add PID file guard |
