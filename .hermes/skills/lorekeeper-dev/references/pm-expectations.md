# PM Expectations

Things Akane (PM) will check on every review:

1. **The ticket file is updated** — root cause documented, not just `status: done`.
2. **Regression test exists** — especially for bugs.
3. **No unrelated changes in the PR** — if you spot other issues, file a new ticket.
4. **Ask early if requirements are unclear** — don't build on assumptions.
5. **Flag scope creep immediately** — if the ticket turns out 3x the work, say so before diving in.

The bar isn't perfection — it's transparency and traceability.
